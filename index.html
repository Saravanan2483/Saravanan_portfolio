AIRA-ORBIS FMS

Autonomous Fleet Management System · Robonetics Automation Solutions LLP · 2024–Present

Node.js Express.js MySQL WebSocket Modbus TCP/RTU SAP WMS Industrial IoT TCP

Production-deployed middleware bridging SAP WMS business intent with a 10+ AMR fleet in a live warehouse. Manages real-time mission orchestration, collision prevention, and inventory synchronization.

Core Architecture

  • Engineered asynchronous, state-driven middleware mapping SAP WMS 'business intent' to low-level AMR actions — FMS acts as a high-fidelity shadow mirror ensuring 100% inventory sync between physical and virtual records.
  • Designed Transaction Orchestrator with journaled sub-tasks: missions auto-resume from exact failure point after network drops or server restarts — zero mission loss.
  • Implemented database-backed Rack Locking preventing race conditions across 10+ concurrent robots; guarantees single-robot exclusive access to each physical pick point.

Algorithmic Innovation

  • Geometry-Aware Stack Resolution Algorithm: recursively detects blocker pallets, executes top-down buffer sequence, retrieves target, then restores stack in correct bottom-up order — eliminates manual rearrangement.
  • Real-time Collision Prevention System (CPS) with priority-based traffic control, configurable safety radii, and predictive path resolution for high-density multi-robot warehouse aisles.
  • WebSocket telemetry engine processing <500ms-loop updates for live pose (X, Y, θ), velocity, battery SOC, and localization confidence across entire AMR fleet simultaneously.

Safety & Simulation

  • Built high-fidelity Simulation Engine to validate complex multi-robot mission flows virtually before physical deployment — reduced on-site commissioning time by ~40%.
  • Fail-Safe Heartbeat monitor auto-halts all AMR operations on WMS connection loss; full Audit Log system for operational traceability and post-incident analysis.
